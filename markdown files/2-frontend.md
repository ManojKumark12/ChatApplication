
---

## Description

### pages
Contains the main UI pages of the application.
---

### common
Contains shared logic and reusable utilities.

Examples:
- Helper functions
- Utility functions
- Shared UI components

Purpose:
- Avoid duplicate code
- Centralize reusable logic

---

### routes
Handles application routing.

Responsibilities:
- Define route paths
- Connect routes to pages

---

### index.css
Contains global CSS styles applied across the application.

Future improvement:
- Convert styles into **CSS modules** for better scalability and maintainability.
